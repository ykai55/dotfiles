# paste

这个 skill 用来安全地读取当前机器的系统剪切板，优先通过 `clip get` 获取文本内容。

## 何时使用

- 用户明确要求读取当前剪切板。
- 用户表示内容已经复制好了，希望你直接使用，例如“我刚复制了一段文本”“直接分析我剪切板里的文本”。
- 用户让你粘贴、查看、总结、分析当前 clipboard / 剪切板内容。

不要在以下场景使用：

- 用户只是让你“复制”某段内容到剪切板。
- 用户没有明确同意读取当前剪切板。

## 工作流

1. 先运行 `clip get`。
2. 如果命令成功且输出非空，按文本内容处理：直接引用、总结，或继续用于后续任务。
3. 如果输出为空，运行 `clip types` 查看剪切板是否有其它 MIME 类型。
4. 如用户明确需要指定 MIME 内容，使用 `clip get --type <MIME>`；二进制内容必须加 `--output <path>`，再按需用 Read 工具读取输出文件。
5. 如果 `clip get` 失败或没有可读文本，明确告诉用户当前剪切板里没有可识别的文本内容。

## 命令参考

- `clip get`: 读取默认文本剪切板内容。
- `clip types`: 列出当前剪切板可用 MIME 类型。
- `clip get --type <MIME>`: 读取指定 MIME 内容。
- `clip get --type <MIME> --output <path>`: 将指定 MIME 内容写入文件，适合二进制内容。

## 响应原则

- 默认先简洁汇报识别到的类型，再给出具体内容或下一步结果。
- 剪切板可能包含敏感信息；除非用户需要完整内容，否则优先摘要而不是整段复述。
- 当剪切板里同时包含多种 MIME 表示形式时，先报告 `clip types` 的结果，再按用户需求选择读取哪一种。

## 示例

用户说：`我刚复制了一段报错，帮我看一下`

建议流程：

1. 运行 `clip get`
2. 读取文本报错内容
3. 基于文本内容继续回答

用户说：`看一下当前剪切板有哪些 MIME 类型`

建议流程：

1. 运行 `clip types`
2. 简洁列出 MIME 类型
3. 如用户要读取某一项，再运行 `clip get --type <MIME>` 或加 `--output <path>`
